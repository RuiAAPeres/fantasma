- Standing worker-tuning principle as of March 15, 2026: favor representative append freshness over backfill throughput when choosing default worker settings. Fantasma's intended traffic is durable SDK-driven mobile append traffic, not operator-driven historical replay. The official iOS SDK persists events in a FIFO SQLite queue, stamps timestamps at enqueue time, and flushes sequentially, so normal client traffic should be append-dominant with very low repair pressure. Treat repair as a bounded guardrail path and backfill as a non-default/operator-only path. A worker-default candidate may be kept if it materially improves representative append freshness while keeping mixed-repair behavior bounded and operationally safe; backfill regressions alone should not block promotion. Revisit this policy only if Fantasma adds client-supplied timestamps, historical import behavior, or other first-class sources of normal-path out-of-order traffic.

## Engineering Ethos

- Fantasma prefers simple, explicit, high-performance data paths.
- We avoid steady-state designs that rely on broad recomputation.
- We narrow feature shape when necessary to preserve predictable performance and operational clarity.
- Correctness still matters, but we should choose correctness models that compose with incremental processing.
